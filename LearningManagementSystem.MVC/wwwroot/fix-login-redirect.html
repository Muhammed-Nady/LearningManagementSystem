?? Fix: Login Redirect Issue

This tool will fix the issue where you're redirected to login even when signed in.

?? Important: You will need to login again after running this fix!

What's Wrong?

Problem: Authentication token is only stored in localStorage, but the server needs it in cookies.

Result: Server thinks you're not logged in ? redirects to login page

The Fix (3 Steps)

1 Clear Old Auth Data

Remove old localStorage and cookies that don't have the token in cookies.

2 Verify Cleanup

Check if localStorage and cookies are cleared.

3 Login Again

After clearing data, login again. The new code will save token to BOTH localStorage AND cookies.

Manual Commands (if needed)

Run these in browser console (F12):

// Clear localStorage localStorage.clear(); // Clear ALL cookies document.cookie.split(";").forEach(c => { document.cookie = c.trim().split("=")[0] + "=;expires=Thu, 01 Jan 1970 00:00:00 UTC;path=/"; }); // Reload location.reload();

After Fix

? Expected Results:
  • Can access /Student/Dashboard without redirect
  • Can access /Student/MyCourses without redirect
  • Token persists for 7 days
  • Server-side authorization works
